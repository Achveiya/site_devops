TD 3 : Orchestration d’Infrastructure

Part 1 : Orchestration de Serveurs avec Ansible

Objectif : Apprendre à provisionner des instances de serveurs (EC2) et automatiser la configuration logicielle.

  • Création de 3 instances EC2 sur AWS.
  • Inventaire dynamique basé sur les tags AWS.
  • Déploiement d’une application Node.js avec PM2.
  • Mise en place d’un Load Balancer Nginx.
  • Mises à jour roulantes (rolling updates).

Part 2 : Orchestration de VM avec Packer et OpenTofu

Objectif : Créer des images immuables et gérer l’infrastructure via le code.

  • Construction d’une AMI avec Packer.
  • Déploiement via OpenTofu avec Auto Scaling Group.
  • Configuration d’un Application Load Balancer (ALB).
  • Mise à jour logicielle avec Instance Refresh.

Part 3 : Orchestration de Conteneurs avec Docker et Kubernetes

Objectif : Gérer le cycle de vie des conteneurs et orchestrer dans le cloud.

  • Conteneurisation avec Docker.
  • Déploiement sur Kubernetes local.
  • Création d’un cluster EKS avec OpenTofu.
  • Stockage des images dans Amazon ECR.
  • Déploiement final avec Load Balancer cloud.

Part 4 : Orchestration Serverless avec AWS Lambda

Objectif : Déployer sans gérer de serveurs (FaaS).

  • Écriture d’une fonction Node.js.
  • Déploiement via OpenTofu.
  • Exposition via API Gateway.
  • Mise à jour rapide du code.

Conclusion

Ce lab compare plusieurs approches d’orchestration, depuis la gestion de serveurs avec Ansible jusqu’au modèle serverless avec AWS Lambda, en passant par les conteneurs et Kubernetes.
